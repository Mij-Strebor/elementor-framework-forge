# AFF Specification — Variables Module

**Version:** 1.2
**Date:** 2026-03-15
**Author:** Jim Roberts / Jim R Forge

---

## 1. Overview

Variables are the primary asset type in AFF. They map directly to CSS custom properties
that are read from and written to the active Elementor kit CSS file. All three variable
sets (Colors, Fonts, Numbers) share a common data model and UI layout; they differ only
in the value types they accept and the controls rendered for each value.

---

## 2. Variable Sets

| Set     | Subgroup key | Category config key  | Value types                      | Has expand panel |
|---------|-------------|----------------------|----------------------------------|-----------------|
| Colors  | `Colors`    | `colorCategories`    | HEX, HEXA, RGB, RGBA, HSL, HSLA | Yes             |
| Fonts   | `Fonts`     | `fontCategories`     | System, Custom                   | No              |
| Numbers | `Numbers`   | `numberCategories`   | PX, %, EM, REM, VW, VH, CH, FX  | No              |

### 2.1 Default categories per set

**Colors:** Branding, Backgrounds, Neutral, Status, Uncategorized (locked)
**Fonts:** Titles, Text, Uncategorized (locked)
**Numbers:** Spacing, Gaps, Grids, Radius, Uncategorized (locked)

`Uncategorized` is always appended automatically at the end of every set. It cannot be
deleted or renamed. All other categories are user-configurable via the Manage Project
modal (§10).

## 3. Variable Data Model

Each variable object is stored in `AFF.state.variables`. The table below shows every
field, its type, its origin in the Elementor v4 kit CSS, and its role within AFF.

| Field          | Type    | Elementor v4 origin                                      | AFF role                                                                 |
|----------------|---------|----------------------------------------------------------|--------------------------------------------------------------------------|
| `id`           | string  | None — generated by AFF                                  | UUID primary key. Empty string for a new variable not yet saved.         |
| `name`         | string  | CSS custom property name from the `:root` block, e.g. `--aff-color-brand-primary` | Stored and displayed with the `--` prefix. The prefix is protected from deletion in the name input. |
| `value`        | string  | The property value from the `:root` block, e.g. `#2C3E50` | The working value the user edits in AFF. May differ from `original_value` when modified. |
| `original_value` | string | Same as `value` at last Sync from Elementor            | Baseline for conflict detection. Updated only on Sync, never on user edit. |
| `format`       | string  | Inferred from the value syntax at parse time             | Display/edit format code — one of `HEX`, `HEXA`, `RGB`, `RGBA`, `HSL`, `HSLA` (Colors), `system`/`custom` (Fonts), or a unit suffix (Numbers). Persisted per variable. |
| `group`        | string  | None — structural                                        | Always `"Variables"` for all variable assets.                            |
| `subgroup`     | string  | None — structural                                        | `"Colors"`, `"Fonts"`, or `"Numbers"`. Routes to the correct panel.      |
| `category`     | string  | None — AFF-only concept                                  | Display name of the assigned category. Denormalized for rendering speed; must be kept in sync when a category is renamed. |
| `category_id`  | string  | None — AFF-only concept                                  | UUID of the assigned category. Authoritative key for category membership. Used to re-resolve `category` name after a rename. |
| `order`        | number  | None — AFF-only concept                                  | Sort position within its category (0-based, ascending). Updated on drag-to-reorder. |
| `status`       | string  | Computed at Sync time                                    | Lifecycle state. See §3.1.                                               |
| `source`       | string  | Determined at import time                                | `"elementor-parsed"` — read from Elementor CSS. `"user-created"` — added manually in AFF. `"imported"` — loaded from an `.aff.json` file. |
| `created_at`   | string  | None — generated by AFF                                  | ISO 8601 timestamp. Set once on creation.                                |
| `updated_at`   | string  | None — generated by AFF                                  | ISO 8601 timestamp. Updated on every save.                               |

> **Note on `original_value`:** This field does not exist in Elementor — it is an AFF
> construct. It stores the value as last read from Elementor so AFF can detect whether
> the user has diverged from the live Elementor state. On first parse it equals `value`.

> **Note on `modified` field (removed):** A separate `modified` boolean appeared in
> earlier drafts. It is superseded by `status`. Do not implement `modified` as a
> standalone field.

### 3.1 Status values

Status is represented visually by a small colored dot in column 2 of the variable row
(§5.1). The dot color provides an at-a-glance signal on every row.

| Status     | Dot color               | CSS token                  | Meaning                                                       |
|------------|-------------------------|----------------------------|---------------------------------------------------------------|
| `synced`   | Green — `#059669`       | `--aff-clr-success`        | Value matches the Elementor kit. No pending commit needed.    |
| `modified` | Gold — `#f4c542`        | `--aff-clr-accent`         | Value changed in AFF but not yet committed to Elementor.      |
| `new`      | Blue — `#3b82f6`        | `--aff-clr-info`           | Created in AFF; does not yet exist in Elementor.              |
| `deleted`  | Red — `#dc2626`         | `--aff-clr-error`          | Marked for deletion. Will be removed from Elementor on next commit. Row is struck through and dimmed. |

The dot tooltip reads `"Status: {status}"` (short form) or the extended text below when
extended tooltips are enabled (§7.3):

| Status     | Extended tooltip text                                                        |
|------------|------------------------------------------------------------------------------|
| `synced`   | "In sync with Elementor — no changes pending"                                |
| `modified` | "Value changed in AFF — commit to update Elementor"                          |
| `new`      | "New variable — not yet in Elementor — commit to add it"                     |
| `deleted`  | "Marked for deletion — commit to remove from Elementor"                      |

## 4. Category Data Model

Categories are stored in `AFF.state.config.colorCategories` (or `fontCategories`,
`numberCategories`). Each category object:

| Field    | Type    | Notes                                                        |
|---------|---------|--------------------------------------------------------------|
| `id`    | string  | UUID, or the slug `"uncategorized"` for the locked default.  |
| `name`  | string  | Display name shown in the left panel and category header.    |
| `order` | number  | Sort order in the left panel (0-based, ascending).           |
| `locked`| boolean | `true` for Uncategorized — cannot be deleted or renamed.     |

### 4.1 Category management rules

1. `Uncategorized` is always present and always last. It is immovable and undeleteable.
2. When a category is deleted, all variables assigned to it are reassigned to
   `Uncategorized` (`category = "Uncategorized"`, `category_id = "uncategorized"`).
3. When a category is renamed, its `id` is preserved. All variables linked by
   `category_id` must have their `category` display-name field updated in the data
   store at rename time so the denormalized field stays consistent.
4. New categories created in the Manage Project modal receive a generated ID
   (`cat-{timestamp}`). The server normalizes the ID on first save.
5. The category list for each set is persisted in the project config via
   `aff_save_config`.

## 6. Colors Expand Panel (Colors only)

Each color variable row has an expand button (▶) that opens an inline expand panel
directly below the row. The panel has three equal-width zones:

| Zone            | Control                                  | Saves on          |
|-----------------|------------------------------------------|-------------------|
| Tints           | `<input type="number" min="0" max="10">` | Debounce 600ms    |
| Shades          | `<input type="number" min="0" max="10">` | Debounce 600ms    |
| Transparencies  | `<input type="checkbox">`                | Immediate toggle  |

Tint/shade/transparency naming convention and algorithm: see
`AFF-Spec-Colors-Amendments.md §Amendment 1`.

The expand panel includes a close button (`×`) in the top-right corner — no
background/border, font-size 22px, font-weight 700, opacity 0.7, hovering to 1.0.

## 7. Tooltip System

AFF uses a single shared tooltip element (`#aff-tooltip`) positioned by JavaScript. The
tooltip always appears above and offset from the triggering element so it is never
obscured by the cursor.

### 7.1 Positioning rule

The tooltip is positioned so that its bottom edge sits at least 10px above the cursor
hotspot at all times. It must never overlap the element it is describing nor sit
underneath the cursor. If the computed position would place the tooltip above the
viewport edge, it flips to appear below the element instead, preserving the 10px
clearance on the opposite side.

### 7.2 Attributes

| Attribute               | Usage                                                        |
|-------------------------|--------------------------------------------------------------|
| `data-aff-tooltip`      | Short tooltip text. Always shown when tooltips are enabled.  |
| `data-aff-tooltip-long` | Extended text. Shown instead of the short text when Extended Tooltips is on. If absent, the short text is shown regardless of mode. |

### 7.3 Behavior

- **300ms delay** before the tooltip appears. The timer resets on any mouse movement.
- **Delegated binding** — a single `mouseover` listener on `document` handles all
  elements, including dynamically created variable rows and category buttons.
- `_tipEl` tracking prevents repeated show/hide calls as the mouse moves over child
  elements (SVG paths, text spans) within the same trigger element.
- The tooltip is hidden immediately on `mouseout` or `focusout` with no delay.

### 7.4 User preferences

The following settings are exposed in the **Options modal** (gear icon, top menu bar),
in a section labelled **"Tooltips"**:

| Setting            | Control in Options modal                | Default | Saved via           |
|--------------------|------------------------------------------|---------|---------------------|
| Show tooltips      | Checkbox — "Show tooltips"               | On      | `aff_save_settings` |
| Extended tooltips  | Checkbox — "Show extended tooltip text"  | Off     | `aff_save_settings` |

**Show tooltips — Off:** `_showTooltip()` returns immediately without rendering the
tooltip element. No tooltip appears anywhere in the UI.

**Extended tooltips — On:** The tooltip displays `data-aff-tooltip-long` content when
that attribute is present on the trigger element. When the attribute is absent, the
short `data-aff-tooltip` text is displayed as normal.

Preferences are loaded asynchronously in `AFF.PanelTop.init()` and stored in the
`_showTooltips` and `_extendedTooltips` properties of `AFF.PanelTop`.

## 8. Data Flow

The diagram below shows the full lifecycle of variable data from its origin in
Elementor through user editing in AFF and back out to Elementor.

```
 ┌─────────────────────┐        ┌───────────────────────┐
 │  Elementor kit CSS  │        │  AFF project file     │
 │  (post-{id}.css)    │        │  (.aff.json)          │
 └──────────┬──────────┘        └──────────┬────────────┘
            │ aff_sync_from_elementor       │ aff_load_file
            │ (reads :root block,           │ (reads full variable
            │  parses CSS custom props)     │  and category state)
            └──────────────┬───────────────┘
                           │ merge into
                           ▼
              ┌────────────────────────┐
              │   AFF.state.variables  │   ← in-memory working store
              │   AFF.state.config     │
              └────────────┬───────────┘
                           │
              ┌────────────▼───────────┐
              │   User edits in AFF    │
              │   (rename, revalue,    │
              │    reorder, add,       │
              │    delete, categorize) │
              └────────────┬───────────┘
                           │
              ┌────────────┴──────────────────────┐
              │                                   │
              ▼                                   ▼
  aff_save_color                        aff_save_file
  (per-variable, fires on every         (full file write, fires
   field change — debounced)             when user presses
                                         "Save Changes")
              │                                   │
              └──────────────┬────────────────────┘
                             │ both write to
                             ▼
              ┌────────────────────────┐
              │  .aff.json on disk     │   ← persistent project file
              └────────────┬───────────┘
                           │
                           │ aff_commit_to_elementor
                           │ (writes modified/new/deleted
                           │  variables back to Elementor
                           │  via the Elementor API)
                           ▼
              ┌─────────────────────┐
              │  Elementor kit CSS  │   ← updated
              │  (post-{id}.css)    │
              └─────────────────────┘
```

**Key points:**

- **Two entry paths** feed `AFF.state.variables`: a live parse from Elementor CSS
  (`aff_sync_from_elementor`) and a file load from a saved project
  (`aff_load_file`). On page load both may run; the merge rule is that the file
  state takes precedence for fields AFF owns (value, category, order, status),
  and Elementor takes precedence for `original_value`.
- **Per-variable saves** (`aff_save_color`) fire on every field edit (debounced) to
  keep the `.aff.json` file continuously up to date.
- **Full-file saves** (`aff_save_file`) write the complete `AFF.state` snapshot and
  are triggered by the "Save Changes" button. They must never run while
  per-variable saves are in flight (§8.1).
- **Commit** (`aff_commit_to_elementor`) is a deliberate, user-initiated action. It
  is the only path by which AFF writes back to Elementor. It does not happen
  automatically on save.

### 8.1 Save guard

`AFF.state.pendingSaveCount` tracks in-flight per-variable AJAX saves. The Save Changes
button is disabled while `pendingSaveCount > 0` to prevent a full-file write from
overwriting in-flight per-variable changes with a stale snapshot.

## 9. PHP Endpoints (Variables)

| Action                   | Method | Description                                          |
|--------------------------|--------|------------------------------------------------------|
| `aff_save_color`         | POST   | Save a single variable's field changes               |
| `aff_add_category`       | POST   | Add a new category to a set (requires `subgroup`)    |
| `aff_delete_category`    | POST   | Delete a category; reassigns its variables to Uncategorized |
| `aff_rename_category`    | POST   | Rename a category by ID                              |
| `aff_reorder_categories` | POST   | Reorder the category list for a set                  |
| `aff_get_usage_counts`   | POST   | Count widget references to each variable name        |
| `aff_commit_to_elementor`| POST   | Write variables to the Elementor kit CSS file        |

All category endpoints accept a `subgroup` POST parameter (`Colors`, `Fonts`,
`Numbers`) to route to the correct category array in the data store.

## 10. Manage Project Modal — Categories Editor

The Manage Project modal (`aff-btn-manage-project`) contains three stacked category
editor panels, one per variable set: **Colors Categories**, **Fonts Categories**, and
**Numbers Categories**.

### 10.1 Layout

Each panel is a labelled section with a single multi-line text area:

```
┌─ Colors Categories ───────────────────────────────────────┐
│ Edit categories as a comma-separated list.                │
│ Uncategorized is always added automatically — do not      │
│ include it here.                                          │
│                                                           │
│ ┌───────────────────────────────────────────────────────┐ │
│ │ Branding, Backgrounds, Neutral, Status                │ │
│ └───────────────────────────────────────────────────────┘ │
└───────────────────────────────────────────────────────────┘

┌─ Fonts Categories ────────────────────────────────────────┐
│ Edit categories as a comma-separated list.                │
│ Uncategorized is always added automatically — do not      │
│ include it here.                                          │
│                                                           │
│ ┌───────────────────────────────────────────────────────┐ │
│ │ Titles, Text                                          │ │
│ └───────────────────────────────────────────────────────┘ │
└───────────────────────────────────────────────────────────┘

┌─ Numbers Categories ──────────────────────────────────────┐
│ Edit categories as a comma-separated list.                │
│ Uncategorized is always added automatically — do not      │
│ include it here.                                          │
│                                                           │
│ ┌───────────────────────────────────────────────────────┐ │
│ │ Spacing, Gaps, Grids, Radius                          │ │
│ └───────────────────────────────────────────────────────┘ │
└───────────────────────────────────────────────────────────┘

                              [ Cancel ]  [ Save ]
```

### 10.2 Default values

| Set     | Default category string (pre-filled) |
|---------|--------------------------------------|
| Colors  | `Branding, Backgrounds, Neutral, Status` |
| Fonts   | `Titles, Text`                       |
| Numbers | `Spacing, Gaps, Grids, Radius`       |

`Uncategorized` is never shown in the text area. It is appended automatically by the
server and always appears last in each set.

### 10.3 Behavior

- The textarea is pre-populated from `AFF.state.config.{set}Categories` when the modal
  opens, with each category name joined by `", "`.
- The user edits the comma-separated string directly. Leading and trailing whitespace
  around each name is trimmed on parse.
- Empty names (e.g. from a trailing comma) are silently discarded.
- Renaming a category by editing its name in the string triggers a rename of any
  category whose name matches the old value. If a name in the saved string does not
  match any existing category ID, it is treated as a new category.
- Removing a name from the string deletes that category. All variables in the deleted
  category are reassigned to `Uncategorized` (§4.1 rule 2).
- On **Save**, the parsed array is sent via `aff_save_config`. `AFF.state.config` is
  updated immediately on success. The active variable panel (if open) is re-rendered
  to reflect the updated category list.
- On **Cancel**, no changes are applied and the modal closes.
